To use the Prometheus client with the Gin framework, you need to use a middleware that collects the metrics. Here is how you can modify the metrics handler:

```go
package main

import (
    "github.com/gin-gonic/gin"
    "github.com/prometheus/client_golang/prometheus"
    "github.com/prometheus/client_golang/prometheus/promhttp"
)

var cpuTemp = prometheus.NewGauge(prometheus.GaugeOpts{
    Name: "cpu_temperature_celsius",
    Help: "Current temperature of the CPU.",
})

func init() {
    // Metrics have to be registered to be exposed:
    prometheus.MustRegister(cpuTemp)
}

func main() {
    cpuTemp.Set(65.3)

    r := gin.Default()

    // Use the promhttp handler as a middleware
    r.GET("/metrics", gin.WrapH(promhttp.Handler()))

    r.Run(":8080") // listen and serve on 0.0.0.0:8080
}
```

In this code, `gin.WrapH(promhttp.Handler())` wraps the Prometheus HTTP handler to a Gin handler. The metrics are then exposed at the `/metrics` endpoint.
